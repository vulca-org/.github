# Contributing to Vulca Systems

Choose the repository that owns the behavior, evidence, integration, or policy
you want to change. Read its README and contribution guidance first.

1. Confirm the repository lane and canonical responsibility.
2. Search existing issues and pull requests for overlapping work.
3. Keep public examples free of credentials, private paths, restricted evidence,
   and customer material.
4. Include focused tests or verification appropriate to that repository.
5. Separate product changes from transfer, rename, visibility, and release work.

Repository-specific guidance takes precedence when it is stricter.
