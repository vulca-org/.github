# Security Policy

## Reporting a vulnerability

Use private vulnerability reporting or a private Security Advisory in the
affected repository. Do not open a public issue, discussion, or pull request
for a suspected vulnerability.

Include the affected repository, revision or release, reproduction conditions,
impact, and the minimum evidence needed to investigate. Do not include customer
data, credentials, private repository content, or unrelated personal data.

If the repository has no private reporting surface, stop and use a previously
verified private maintainer route. Do not substitute a public issue.

## Scope

Each repository defines its supported releases and response expectations. This
policy supplies a reporting boundary, not a universal remediation SLA.
