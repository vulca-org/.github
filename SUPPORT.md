# Support

Use the affected repository's documented issue or discussion surface for
public usage questions, reproducible defects, and feature proposals. Include
the repository version or revision, environment, expected behavior, observed
behavior, and a minimal public-safe reproduction.

Do not use support channels for vulnerabilities, credentials, customer data,
private repository content, or restricted evidence. Follow `SECURITY.md` for
vulnerabilities and use a verified private route for other sensitive material.

Vulca Systems does not provide one organization-wide runtime SLA. Each
repository states its own maintenance and support boundary.
